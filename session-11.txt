NoSQL
=======
data is un sctructured
useful when huge amounts of data
data is stored as documents
generally used for product catalouges, Real-time analytics, etc but not for transactions

Redis
=======
in-memory(RAM) database
HD contains data -> places in RAM -> Processor
for every application along with HD we should allocate sufficient RAM also
redis is a key-value pair  database
apps checks -> cache -> if not available in cache -> database

Asynchronous 
===========
ActiveMQ, IBM MQ, RabbitMQ, Kafka, etc...

synchronous -> request and wait -> http
asynchronous -> fire and forget  -> mail, whatsapp, etc..

Prodcuer -> Who is sending message
Consumer -> Who is consuming message

in asynchronous
No waiting, good user experience
Scalability
Reliability
Loose Coupling -> gives request to other system -> doesn't waits for response
Tight Coupling -> complete dependency -> with other system -> waits for response -> performance slows down 
tight coupling with the same system is okay

when there is urgency use synchronous otherwise use asynchronous

Queue management -> FIFO (first in first out) -> one to one

publish & subscribe
