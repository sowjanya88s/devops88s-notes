installing a package using ansible
03-playbook.yaml
=================
- name: install and run nginx
  hosts: web
  become: yes	//running as root user
  tasks:
  - name: install nginx
    ansible.builtin.package:   //installs nginx by using package module
      name: nginx
      state: present

  - name: start nginx
    ansible.builtin.service:	//starting nginx using service module
      name: nginx
      state: started
      enabled: yes 	//enables and starts the service at a time

command to execute -- ansible-playbook -i inventory.ini -e ansible_user=ec2-user -e ansible_password=DevOps321 03-playbook.yaml

while gathering facts -- it understands the os -- and runs os related command to install nginx
that is how ansible acts as heterogeneous

defining variables in ansible

if we give hosts  & connection as local there is no need to provide username and password also
in below program variable are defined at play level so they are called play level variables

04-vars.yaml
============
- name: play level variables
  hosts: local
  connection: local
  vars:
    COURSE: "DevOps with AWS"
    DURATION: "140HRS"
    TRAINER: "Sivakumar Reddy"
  tasks:
  - name: print variables
    ansible.builtin.debug:
      msg: "Course is: {{ COURSE }}, Trainer is: {{ TRAINER }}"  //to print variable {{ <var-name> }}

comand to execute -- ansible-playbook -i inventory.ini 04-vars.yaml

output --
msg: "Course is: DevOps with AWS , Trainer is: Sivakumar Reddy "

now we will see task level variables , we can mention them in task
if a variable is defined at play level and task level -- which value will be printed
this is called as overriding
if we does not override what we mentioned at play level that will be printed

variables defined at play level -- will be assigned to all taks -- called inheritance

we can take info from user also 
07-vars.yaml
============
- name: print username and password
  hosts: local
  connection: local
  vars_prompt:  // to prompt list of variables
  - name: username
    prompt: Please enter your username 	//prompt text
    private: false 	//not confidential

  - name: password
    prompt: please enter your password
    private: true	//confidential -- does not shows on screen while entering
  
  tasks:
  - name: print info
    ansible.builtin.debug:
      msg: "username is: {{ username }}, password is: {{ password }}"


comand to execute -- ansible-playbook -i inventory.ini 07-vars.yaml
