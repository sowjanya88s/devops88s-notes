how to call another role
we have 2 options : import_role & include_role

import_role vs include_role
===========================
1. tags applied to import_role will be automatically applied to all tasks inside it
2. tags applied to include_role will not be cascaded
3. import role usually parse the playbook before execution, it is static
4. include role is dynamic , it will directly execute the playbook, it will not parse like import_role
5. condition applied to import_role is applied to all tasks, include_role does not cares the condition applied to it
6. u can't use loops in import_role , u can use loops in include_role

parallel execution
==================
ansible supports parallel execution means it can connect to multiple servers at a time it can connect upto 100 servers

client we r using to connect should be powerful (its config should be high)

consider that we have 17 frontend servers
to check 17 servers ips
command -- ansible-inventory -i frontend.aws_ec2.yaml --list

if we try to execute ansible-playbook to configure frontend 
command to execute -- ansible-playbook -i frontend.aws_ec2.yaml -e component=frontend roboshop.yaml

ansible at a time by default can connect to 5 servers . these are called as forks. we can control this -- for that mention in ansible.cfg
===========
forks = 8

means each task is targeted on 8 servers, single playbook execution because we have 17 servers 
if u have higher server capacity it will connect to servers batchwise ( in first batch -- 8 , next batch -- 8 like that)
in this process response time might differ

serial
======
consider we have given serial = 5
from 8 forks -- takes 5 first
runs entire playbook
next runs on 3 servers
it does not waits for task completion , it makes ready on remaining servers (called as rolling update)
making changes in all servers at a time is not correct because if there is any error all servers gets down
 
we have mention serial in playbook
ansible-roboshop-roles -- roboshop.yaml (mention it above roles)
=======================================
- name: "configuring {{ component }}"
  hosts: "{{ component }}"
  become: yes
  serial: 5
  roles:
  - "{{ component }}"

now the playbook will be executed in 5 servers and then executes in 3 servers

next see the config of frontend
command to execute -- ansible-playbook -i frontend.aws_ec2.yaml -e component=frontend roboshop.yaml
