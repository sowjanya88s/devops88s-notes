Terraform
==========
terraform is a popular infrastructure as a code tool
there are alternatives for terraform. they are cloud formation (in AWS) , Arm (in Azure), pulumi

pulumi is a developer related tool & in that we need to write the code
terraform is very easy when compared to pulumi

advantages of terraform
=========================
terraform can connect to multiple clouds , and it can connect to multiple tools

why should we use infra as a code tool??

what are the problems when we create infra manually ?
1. version control

if we configure applications manually there will be no version control. if there is no version control , u can't have history of our infra
if we maintain infra manually , if anybody did changes then troubleshooting will become difficult

if we maintain our infra as code, then it will be easy to restore
if we maintain our infra as code , we can troubleshoot -- we can track who did that & we can review the code changes
if the code changes are not good , we can always go back to previous versions

2.consistent infra

general problem -- the code works in dev but it does not works in prod

here DEV , UAT , PROD --if we are taking one application in multiple env's -- and we are using same infra in all env's -- if that infra works in dev -- then we can say it will definitely works in prod also

maintaining consistent infra in all environments manually becomes difficult -- but if we use infra as a code -- then we can apply the same code in DEV, UAT, PROD

3. CRUD -- inventory management

if we give one aws account to one person -- we told him to check what services are running in that account -- doing this task is difficult
the services which are created one hour or one minute before will not be reflected in billing

if we give one aws account -- with only read access -- in that u can only create infra by using terraform code -- by seeing terraform code we can tell what are the services we are running in aws account

by using terraform we can create , read , update , delete the infra by using terraform

4. cost optimisation 
doing infra creation & update & deletion manually it will takes hours of time 

by using terraform we can do infra creation & update & deletion within minutes

whenever useful we will create infra using terraform (means in working hours) & in non working hours we can delete the infra created in dev & UAT

5. dependency management

to create ec2 instance -- we need security group -- we can manage those dependancies in our code  

6. reusable infra 

whenever we write code -- options required for code reuse are available in tearraform -- terraform modules

lets see creation of security group
ingress -- inbound rules
egress -- outbound rules
all protocols , all traffic is allowd in inbound & outbound rules

ec2.tf
=======
resource "aws_security_group" "allow_tls" {
  name        = "allow-all-terraform" # this is for AWS account
  description = "Allow TLS inbound traffic and all outbound traffic"

  egress {
    from_port        = 0
    to_port          = 0
    protocol         = "-1"
    cidr_blocks      = ["0.0.0.0/0"]
    ipv6_cidr_blocks = ["::/0"]
  }

  ingress {
    from_port        = 0
    to_port          = 0
    protocol         = "-1"
    cidr_blocks      = ["0.0.0.0/0"]
    ipv6_cidr_blocks = ["::/0"]
  }

  tags = {
    Name = "allow-all-terraform"
  }
}

ec2.tf
======
resource "aws_instance" "example" {
  ami           = "ami-0220d79f3f480ecf5"
  instance_type = "t3.micro"
  vpc_security_group_ids = [aws_security_group.allow_tls.id]

  tags = {
    Name = "terraform-state-demo"
    Project = "roboshop"
  }
}

resource "aws_security_group" "allow_tls" {
  name        = "allow-all-terraform" # this is for AWS account
  description = "Allow TLS inbound traffic and all outbound traffic"

  egress {

    to_port          = 0
    protocol         = "-1"
    cidr_blocks      = ["0.0.0.0/0"]

  }
