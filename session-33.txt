remote sate
===========
so in colloboration env, state should be stored in remote like s3 so that we can prevent errors and duplicates , it should be locked , parallel modifications should not be allowed
if a person work is actively creating , updating and destroying the state file should be locked

now lets use remote state
in infra parallel changes are not allowed
because if one process is running , modifications are under progress , now allowing 2nd person to make changes in the same state file --  there will be lot of confusion -- so terraform blocks modifying state files by multiple persons at one time

s3
===
open aws console -- s3 -- create bucket -- bucket name should be universally unique -- in aws
acls'e enabled
block all public access
create bucket 

create one repo -- remote-state -- provider.tf
in provider.tf only we should tell terraform -- to use state file from s3
search -- remote state aws terraform with lock -- for lock code -- u can see terraform native locking 

provider.tf
===========
terraform {
  required_providers {
    aws = {
      source = "hashicorp/aws"
      version = "6.33.0" # Terraform AWS provider version
    }
  }

  backend "s3" {
    bucket  = "remote-state-aws-88s" # Replace with your unique bucket name
    key     = "remote-state.tfstate"  # name of the state file
    region  = "us-east-1"
    encrypt = true
    use_lockfile   = true  # to lock state file while applying infra
  }
}

provider "aws" {
  region = "us-east-1"
}

when u don't use lock file -- multiple persons can try to access state file -- state file will be corrupted

now run terraform init -- successfully configured the backend s3 terraform will automatically use this backend unless the backend configuration changes
