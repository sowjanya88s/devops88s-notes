in our aws account we have to create vpc first -- in that create public & private & data base subnets

request from frontend goes to components , we need load balancer between while sending requests from frontend to components because upto now we have configured only one catalogue instance , to improve scalability in our project -- when traffic increaes -- we need multiple catalogue servers
when no.of servers increaese -- we need load balancer compulsarily
for ex cart depends on catalogue , and we have 5 catalogue servers , load balancer checks which catalogue server is free & then decides which catalogue server , the cart should connect to
before frontend also we will place one load balancer 
the load balancer before frontend is public -- we use https here
& the load balancer before components is private (internal) -- we use http here
here we will configure security group for each and every component 

project infra -- one time infra (vpc , sg , LB , databases) -- changes very rarely
application infra -- keeps changing (sg rules , components) -- based on traffic no.of catalogue servers may increase or decrease & to increase security we may modify sg rules

terraform-aws-sg -- main.tf
===========================
resource "aws_security_group" "main" {
  name        = "${var.project}-${var.environment}-${var.sg_name}"
  description = "Allow TLS inbound traffic for ${var.project} in ${var.environment} for component ${var.sg_name}"
  vpc_id      = var.vpc_id

  egress {
    from_port        = 0
    to_port          = 0
    protocol         = "-1"
    cidr_blocks      = ["0.0.0.0/0"]
  }
  
  tags = merge(
    var.sg_tags,
    local.common_tags,
    {
        Name = "${var.project}-${var.environment}-${var.sg_name}"
    }
  )
}
