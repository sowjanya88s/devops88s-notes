CDN(cloudfront)
===============
amazon prime , Netflix are video streaming platforms
generally Netflix servers are in US
when u watch Netflix content from india -- content should continuously flow from us to india -- there will be some latency & unnecessary traffic/bandwidth

we have discussed one topic in previous classes , application does not always hits DB always , application checks cache first , if cache is not available then it hits DB

in video/content streaming platforms -- browser does not always connect to origin server ,  there is something called CDN(content delivery network)

CDN -- it is a network of storage servers (also called as edge location) across the globe used to cache the content. there will be an origin server , 
first application hits edge location , if data is not there the it gets data from origin & saves it in edge location so that subsequent requests get data from edge location

AWS has 400 edge locations around the world
search -- aws edge location in india 
Mumbai , new delhi , Chennai , Kolkata, bangalore , Hyderabad , pune

consider that a new movie came , when the 1st person try to watch the movie from hyd , browser checks the nearest the edge location(hyd) for the content , if the content is not available in edge location, then it gets the content from origin server and saves the data in edge location. then if anybody tries to watch the content from hyd the browser fetches the content from hyd edge location , by this process speed increases when anyone tries to access the content -- this is how cache works in CDN


we can control the type of the content -- 
there are 4 types of http requests
GET POST UPDATE DELETE
in 4 options -- GET is the safest operation
GET (reading info) -- can be cached
POST UPDATE DELETE -- generally shouldn't be cached
blind rule -- info reading can be cached (for ex: product details -- images , description , faq , rating, reviews)
images , description , faq -- not frequently changing -- we can cache this things for much time
rating & reviews -- frequently changing -- we can cache this things for less amount of time 
we can also control TTL(time to live)

.js , .css , images -- can be cached 

invalidation 
============
when u applied changes in ur application & u have deployed those changes -- we should delete the cache , otherwise old content will be displayed to the users

when new deployment /release happens usually we should invalidate all the caching content , it deletes the data from all the edge servers. content will be downloaded again from origin to edge locations

whenever there is a release -- invalidation is the better option

the origin can be ec2 server or s3 bucket or Load balancer
u should decide which requests should be cached & which requests should be invalidated

now user directly hits CDN , we attach this CDN to route53 , CDn forward the requests to frontend LB (cloudfront takes frontend ALB as origin because content comes from frontend LB)

main.tf
========
resource "aws_cloudfront_distribution" "roboshop" {
  origin {
    # https://frontend-dev.daws88s.online
    domain_name              = "frontend-${var.environment}.${var.domain_name}"
    origin_id                = "frontend-${var.environment}.${var.domain_name}"

    custom_origin_config  {
        http_port              = 80 // Required to be set but not used
        https_port             = 443
        origin_protocol_policy = "https-only"
        origin_ssl_protocols   = ["TLSv1.2", "TLSv1.1"]
    }
  }

  enabled             = true
  is_ipv6_enabled     = false
  
  # CDN URL https://roboshop-dev.daws88s.online
  aliases = ["${var.project}-${var.environment}.${var.domain_name}"]

  default_cache_behavior {
    allowed_methods  = ["DELETE", "GET", "HEAD", "OPTIONS", "PATCH", "POST", "PUT"]
    cached_methods   = ["GET", "HEAD"]
    target_origin_id = "frontend-${var.environment}.${var.domain_name}"

    viewer_protocol_policy = "https-only"
    cache_policy_id = local.cachingDisabled
  }

  # Cache behavior with precedence 0
  ordered_cache_behavior {
    path_pattern     = "/media/*"
    allowed_methods  = ["GET", "HEAD", "OPTIONS"]
    cached_methods   = ["GET", "HEAD", "OPTIONS"]
    target_origin_id = "frontend-${var.environment}.${var.domain_name}"

    viewer_protocol_policy = "https-only"
    cache_policy_id = local.cachingOptimized
  }

  # Cache behavior with precedence 1
  ordered_cache_behavior {
    path_pattern     = "/images/*"
    allowed_methods  = ["GET", "HEAD", "OPTIONS"]
    cached_methods   = ["GET", "HEAD", "OPTIONS"]
    target_origin_id = "frontend-${var.environment}.${var.domain_name}"

    viewer_protocol_policy = "https-only"
    cache_policy_id = local.cachingOptimized
  }

  price_class = "PriceClass_All"

  restrictions {
    geo_restriction {
      restriction_type = "none"
      #locations        = ["US", "CA", "GB", "DE"]
    }
  }

  tags = merge(
    {
        Name = "${var.project}-${var.environment}-frontend"
    },
    local.common_tags
  )

  viewer_certificate {
    acm_certificate_arn = local.acm_certificate_arn
    ssl_support_method  = "sni-only"
  }
}

resource "aws_route53_record" "cdn" {
  zone_id = var.zone_id
  name    = "${var.project}-${var.environment}.${var.domain_name}"
  type    = "A"
  
  # CDN details
  alias {
    name                   = aws_cloudfront_distribution.roboshop.domain_name
    zone_id                = aws_cloudfront_distribution.roboshop.hosted_zone_id
    evaluate_target_health = true
  }
  allow_overwrite = true
}

}

locals {

        Project = var.project
        Environment = var.environment
        Terraform = "true"
    }
}
