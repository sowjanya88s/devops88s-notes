docker ps

command to build images--
docker compose build

command to build a particular image -- 
docker compose build mongodb

after image rebuild use below command to start container --
docker compose up -d  
by running above command -- if images available in docker hub related to roboshop project will be used to run containers

to build images along with container creation --
docker compose up -d --build

to stop containers --
docker compose down

command to remove all images --
docker rmi -f `docker images -aq`


let see MySQL , we r using MySQL 8 version
open docker hub -- search MySQL 8
we can load data from MySQL
download shipping code & place db folder in MySQL folder
in db folder -- schema.sql is not needed so delete it
.sh , .sql files will be executed automatically when we place files in /docker-entrypoint-initdb.d
we need to set root password
mention root password in ENV -- that will be set as root password

create repo -- MySQL -- 

Dockerfile
==========
FROM MySQL:8.0
ENV MYSQL_ROOT_PASSWORD="Roboshop@1"
COPY db/ /docker-entrypoint-initdb.d


git pull
docker compose build mysql

lets see shipping
create repo -- shipping 
copy pom.xml & src folder into shipping
with maven , java will be installed automatically
in documentation 4 version is used for maven , its the latest in docker , so use 3.9 instead

Dockerfile
===========
FROM maven:3.9.14
EXPOSE 8080
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package
RUN mv target/shipping-1.0.jar shipping.jar
ENV CART_ENDPOINT="cart:8080"
    DB_HOST="MySQL"
CMD ["java", "-jar", "shipping.jar"]


git pull
docker compose build shipping

lets see RabbitMQ
goto dockerhub -- search for rabbitmq-3 version
we have to create username & password -- mention them in env variables
there is no customization at the image level
mention in docker-compose.yaml
if we mention environment variables at Dockerfile -- we need to rebuild the image , if there are any changes
if we mention env variables at docker compose (at run time) -- no need to rebuild the image , change the values when u need then restart the container
building the image is expensive process
