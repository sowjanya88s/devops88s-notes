docker layers
=============
clone roboshop-docker repo & build the catalogue image --

how docker handles images ?
it uses layering approach

to see the layers while building the image , use command --
DOCKER_BUILDKIT=0 docker build -t user:1.0.0 --rm=false --no-cache .

when u give docker build 
it take first instruction from this file -- 
FROM node:20  -- create container from this image -- login to that intermediate container & runs 2nd instruction (i.e WORKDIR /app)
 -- creates image by this -- runs container from this 
runs next instruction COPY package.json . -- creates image from this -- runs container by this image -- runs next instructions in container -- like that it build the image

docker  creates the intermediate containers & intermediate images by each & every instruction -- removes them
it just stores the final image & container

summary
=======
docker creates intermediate container for every instruction
it will run next following instruction inside this container -> create image out of it
then it creates another intermediate container from that image, run the next instruction -> create image out of it

so for every instruction new layers are created -> docker exports all these layers into hub while pushing the image

we must keep frequently changing instructions at the bottom of Dockerfile to make the build process fast and storage

while writing docker file , code (js files) changes continuously , place those instructions at bottom
place not frequently changing instructions at top (like WORKDIR , EXPOSE , ENV)

make some changes in .js file (like write one comment in that file) & try to build the image again -- docker rebuilds the image from that js instruction & all above layers will be cached

only changed layers will be pushed to docker hub
