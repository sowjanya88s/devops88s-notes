pod
===
pod is smallest deployable unit in k8s , it is like container but a pod can have multiple containers , containers inside pod share same network and storage

search -- k8s pod

here spec means specifications , in containers we have to mention name of the container & the image we r using to run this container & we can mention the port for the container but it is optional because by default all images opens some default port(for ex: nginx opens 80 port bydefault). in containers we can mention list of containers

resources
=========
containers consumes the resources from node according to its needs

consider a node contains two pods , pod1 has one problem so it started consuming all resources of node , in this case pod2 does not have resources -- problem comes here
we need to always restrict the resources to the pod .. if it gets more requests we can do autoscaling

here also we will start the pod with minimum resources & we will scale them upto maximum level. we will mention resource units.
 
search -- k8s pod resources
1cpu = 1000 milli

in soft link -- requests -- we will mention minimum limits of memory (RAM) & cpu (disk)
100 mi - 1/10 th of 1 cpu
in hard link -- limits -- we will mention maximum limits of memory (RAM) & cpu (disk) -- container can't extend its resource beyond this limits
in projects we have to mentioning this mandatory
to prove this practically we need to apply some stress

06-resources.yaml
=================
apiVersion: v1
kind: Pod
metadata:
  name: resources-demo
spec:
  containers:
  - name: nginx
    image: nginx
    resources:
      # soft limit
      requests:
        memory: "64Mi" # RAM
        cpu: "100m" # CPU
      # hard limit
      limits:
        memory: "128Mi" # RAM
        cpu: "150m" # CPU

kubectl apply -f 06-resources.yaml
kubectl get pods

configmap
=========
You can write a Pod spec that refers to a ConfigMap and configures the container(s) in that Pod based on the data in the ConfigMap. The Pod and the ConfigMap must be in the same namespace.
we can mention key value pairs , we can mention one key & multiple values in this

service
========
consider that we have 2 pods , how can they communicate with each other , they can communicate using ip addresses but they are temporary
so we can use k8s service to achieve
1. pod to pod communication through DNS, service name works as DNS here
2. load balancing

consider cart tries to reach -- catalogue
if we have multiple (cart) resources we should always use load balancer
pod1 can contact pod2 through service
pod1 -- service -- pod2 , pod3 , pod4
based on round robbin way -- service allocates anyone pod to connect with pod1

we have multiple types of services
1. cluster ip
2. node port
3. load balancer

lets see cluster ip now

cluster ip
===========
in this method , we have to mention spec , here mention selectors , give labels (labels are used as selectors) , to which pod we want to attach the service , mention that pod related labels here

by this service will be created & finds pod with mentioned labels & attaches that pod to its team
mention port info of pod & container port
make sure that particular pod is running in same namespace

12-service.yaml
================
apiVersion: v1
kind: Service
metadata:
  name: nginx
spec:
  selector:
    # labels are used as selectors
    project: roboshop
    component: frontend
    environment: dev
  ports:
  - protocol: TCP
    port: 80 # servie port
    targetPort: 80 # container port

commands --
kubectl apply -f 12-service.yaml
kubectl get svc
kubectl describe svc nginx
we can see end points -- nothing but pods it connected to & port
