RDBMS -> relational database management server
MySQL  , oracle, PostgreSQL, mssql comes under RDBMS
the data will be in tables in these RDBMS databases
there will be relations between tables by using primary key and foreign key
if we place all data in one table while querying it will be difficult(maintenance issue)
for example 
user table
id name mobilenumber 
id -- primary key
payments
userid id amount date
userid -- primary key
id -- foreign key

dnf module list nodejs -- lists multiple versions of nodejs by searching all repos

we don't know what is the default version so use disable command to disable the default version

dnf module disable nodejs -y

dnf module enable nodejs:20 -y

dnf install nodejs -y

by this nodejs 20 will be installed because we enabled it

by installing nodejs its dependancy packages will be updated

after nodejs installation try to login to the same server again it will throw an error
reason is  ssh depends on openssl (updated while installing nodejs)and openssh

sshd -t
openssl version mismatch

so we need to update openssh to avoid ssh break

dnf update -y openssh openssh-server openssh-clients

to start that we have to write a service file in /etc/systemd/system

here in conf file of backup we mention
[Unit]

[Service]
User=expense
Environment=DB_HOST="<MYSQL-SERVER-IPADDRESS>"
ExecStart=/bin/node /app/index.js
SyslogIdentifier=backend

[Install]
WantedBy=multi-user.target
