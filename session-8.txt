Forward Proxy
==============
1. hiding client identity (ex: vpn)
2. forward proxy sits in between client and server
3. client aware that they r using forward proxy
4. VPN, content filtering, traffic monitoring
5. Can be used as caching also.

content filtering -- in companies they will restrict ur access of sites at vpn level and vpn logs tour browsing history
 

Reverse Proxy
=============
1. Hiding server identity (users thinks that response is coming from frontend but response actually comes from backend or application server)
2. Server Aware
3. for Security reasons and queue management they use reverse proxy
4. useful for Load balancing and queue management
5. used for SSL/TLS/HTTPS termination

https -- encrypts the data for security reasons
once request comes to our env (server) https is not needed
by https also fetching data becomes slow
we implement https at entry point of our env(i.e front end ) -- this is called ssl/tls/https termination
frontend to backend , backend to db -- here we don't use https
to fastly fetch the data from application server or db server we ignore https , app server and db server are in our environment and they will be in reverse proxy
